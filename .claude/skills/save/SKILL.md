# save: файлирование разговоров в wiki

Хорошие ответы и инсайты не должны исчезать в чат-истории. Этот скилл берёт то, что только что обсуждалось, и файлирует это как постоянную wiki-страницу.

Wiki накапливает. Сохраняй часто.

---

## Команда

| Команда | Поведение |
|---|---|
| `/save` | Спросить название, потом сохранить |
| `/save [название]` | Сохранить с указанным названием (без вопроса) |

---

## Решение по типу заметки

Определи лучший тип из содержания разговора:

| Тип | Папка | Когда использовать |
|---|---|---|
| `synthesis` | `wiki/questions/` | Многошаговый анализ, сравнение, ответ на конкретный вопрос |
| `idea` | `wiki/ideas/` | Объяснение или определение идеи, паттерна, фреймворка |
| `source` | `wiki/sources/` | Саммари внешнего материала, обсуждённого в сессии |

Если пользователь указал тип — используй его. Если нет — выбери по содержанию. При сомнении используй `synthesis`.

---

## Workflow сохранения

1. **Просканируй** текущий разговор. Идентифицируй самое ценное содержимое для сохранения.
2. **Спроси** (если название не указано): "Как назвать эту заметку?" Держи имя коротким и описательным.
3. **Определи** тип заметки по таблице выше.
4. **Извлеки** релевантное содержимое из разговора. Перепиши в декларативном настоящем времени (не "пользователь спросил", а само содержимое).
5. **Создай** заметку в правильной папке с полным frontmatter.
6. **Собери ссылки**: идентифицируй любые wiki-страницы, упомянутые в разговоре. Добавь их в `related` во frontmatter.
7. **Обнови** `wiki/index.md`. Добавь новую запись в начало релевантной секции.
8. **Допиши** в `wiki/log.md`. Новая запись СВЕРХУ:
   ```
   ## [YYYY-MM-DD] save | Заголовок заметки
   - Тип: [тип заметки]
   - Расположение: wiki/[папка]/Заголовок заметки.md
   - Из: разговор о [краткое описание темы]
   ```
9. **Обнови** `wiki/cache.md` с учётом нового добавления.
10. **Подтверди**: "Сохранено как [[Заголовок заметки]] в wiki/[папка]/."

## Шаблон frontmatter

```yaml
---
type: <synthesis|idea|source>
title: "Заголовок заметки"
created: YYYY-MM-DD
updated: YYYY-MM-DD
tags:
  - <релевантный-тег>
status: in-progress
related:
  - "[[Любая упомянутая wiki-страница]]"
sources:
  - "[[.raw/source-если-применимо.md]]"
---
```

Для типа `synthesis` (если это ответ на вопрос), добавь:
```yaml
question: "Оригинальный запрос."
answer_quality: solid
```

## Стиль письма

- Декларативное, настоящее время. Пиши знание, не разговор.
- Не: "Пользователь спросил про X, и Claude объяснил..."
- Да: "X работает через Y. Ключевой инсайт — Z."
- Включай весь релевантный контекст. Будущие сессии должны прочесть страницу с нуля и понять.
- Линкуй каждую упомянутую идею, сущность или wiki-страницу через wikilinks.
- Цитируй источники где применимо: `(Источник: [[Страница]])`.

---

## Что сохранять, а что пропускать

Сохранять:
- Не очевидные инсайты или синтез
- Анализы, которые потребовали значительных усилий
- Сравнения, на которые скорее всего будут ссылаться снова
- Результаты исследований

Пропускать:
- Механические Q&A (lookup-вопросы с очевидными ответами)
- Шаги настройки, уже задокументированные
- Временные debug-сессии без долгосрочного инсайта
- Всё, что уже есть в wiki

Если уже есть в wiki — обнови существующую страницу вместо создания дубля.
