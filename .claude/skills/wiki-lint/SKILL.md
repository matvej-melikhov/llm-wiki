# wiki-lint: проверка здоровья wiki

Lint имеет три режима, отличающихся объёмом работы и правом на правки. Все три читают/пишут `wiki/meta/lint-state.md` — счётчик и timestamp последнего успешного прохода.

---

## Режимы

| Режим | Триггер | Правит | Когда |
|---|---|---|---|
| **skip** | `/lint` на чистом состоянии | — | если max mtime страниц ≤ last_full_audit и issues_remaining = 0 |
| **auto-fix** | конец `/ingest` (Phase 8) или `/lint --auto-fix` | детерминистические правки | автоматически после ingest |
| **full audit** | `/lint` или `/lint --force` | ничего (пишет отчёт) | вручную, периодически или принудительно |

---

## Команда и флаги

| Команда | Поведение |
|---|---|
| `/lint` | Сначала skip-check; если не чисто или есть открытые проблемы — full audit |
| `/lint --force` | Игнорировать skip-check, всегда full audit |
| `/lint --auto-fix` | Применить детерминистические правки + отчёт по остатку |

---

## Кэширование состояния (skip-check)

`wiki/meta/lint-state.md` хранит результат последнего успешного прохода:

```yaml
---
type: meta
title: "Lint State"
updated: 2026-04-29T15:45:00
last_full_audit: 2026-04-29T15:45:00
files_checked: 11
issues_auto_fixed: 3
issues_remaining: 0
---

# Состояние lint

Последний полный аудит: 2026-04-29 15:45.

Wiki в чистом состоянии: повторный lint пропускается до изменений.

## Открытые проблемы

_Нет._
```

**При входе в `/lint`:**

1. Прочитать `wiki/meta/lint-state.md`. Если файла нет — пропустить skip-check, идти на full audit.
2. Найти максимальный `mtime` среди файлов:
   ```bash
   find wiki -name '*.md' \
     -not -path 'wiki/meta/lint-state.md' \
     -not -path 'wiki/meta/lint-report-*.md' \
     -printf '%T@\n' | sort -rn | head -1
   ```
3. Сравнить с `last_full_audit` из state-файла.
4. Если max-mtime ≤ last_full_audit И `issues_remaining: 0` → вывести:
   ```
   Wiki не менялась с последнего аудита (2026-04-29 15:45). Чисто. Пропускаю.
   Используй `/lint --force` для принудительной перепроверки.
   ```
   и завершить.
5. Иначе — продолжать с full audit (если выставлен `--auto-fix`, работать в auto-fix режиме).

После любого успешного прохода (auto-fix или full audit) — **перезаписать** `lint-state.md` с актуальным timestamp и счётчиками.

---

## Auto-fix режим

Применяется автоматически в конце `/ingest` (Phase 8). Цель — закрыть детерминистические нарушения схемы сразу, не отвлекая пользователя.

**Правится автоматически (без подтверждения):**

| Нарушение | Правка |
|---|---|
| `status` вне enum (`stable`, `draft`, `done` и т.д.) | заменить на `in-progress` |
| `status:` присутствует у `type: entity` | удалить поле |
| Поля `title`, `complexity`, `first_mentioned` на не-meta страницах | удалить поля |
| Lowercase теги (`ml`, `rl`, `alignment`, `nlp`) | поднять регистр согласно схеме |
| Inline-формат `tags: [a, b, c]` | переписать в блочный YAML |
| `[[raw/...]]` в теле страницы (вне frontmatter) | удалить ссылку |
| Секции `## Источники` / `## Источники упоминания` содержащие только `[[raw/...]]` | удалить секцию |
| `[[raw/X.md]]` (с расширением) в `sources` | заменить на `[[raw/X]]` |

**Не правится автоматически (выводится пользователю):**

- сироты (нужно решение: слинковать или удалить)
- мёртвые ссылки (нужно создать страницу или убрать ссылку)
- пропущенные концепции (предложение, не баг)
- устаревшие утверждения (требует ревью)
- противоречия

**Вывод после auto-fix:**

```
Auto-fix: поправлено 3 проблемы (status: stable→in-progress на 9 страницах,
теги переведены в правильный регистр, удалены раw-ссылки из 2 секций "Источники").

Осталось требующих ревью: 2.
- 🔴 dead link: [[SFT]] упоминается в [[RLHF]], страницы нет
- 🟡 пропущена: концепция "GAE" встречается на 3 страницах без своей wiki-страницы

Lint state обновлён: 2026-04-29T15:45.
```

Если есть нерешённые проблемы — детальный отчёт сохраняется в `wiki/meta/lint-report-YYYY-MM-DD.md`. Если auto-fix закрыл всё (`issues_remaining: 0`) — отчёт не создаётся, только обновляется `lint-state.md`.

---

## Full audit (manual /lint)

Полный проход по всем проверкам без правок. Создаёт детальный отчёт `wiki/meta/lint-report-YYYY-MM-DD.md`.

После аудита спрашивает: "Хочешь применить безопасные правки сейчас? (это эквивалент `--auto-fix`)" Если да — выполняет auto-fix.

Затем обновляет `lint-state.md`.

---

## Проверки

| # | Проверка | Auto-fix? |
|---|---|---|
| 1 | Сироты (страницы без входящих wikilinks) | нет |
| 2 | Мёртвые ссылки | нет |
| 3 | Устаревшие утверждения | нет |
| 4 | Пропущенные страницы (упомянуты на нескольких страницах без своей) | нет |
| 5 | Отсутствующие перекрёстные ссылки (упоминание сущности без wikilink) | да (если страница точно существует) |
| 6 | Пробелы во frontmatter (отсутствуют обязательные поля) | да (добавить с дефолтами) |
| 7 | Поля старой схемы (`title`, `complexity`, `first_mentioned`, `status` на entity) | да (удалить) |
| 8 | `status` вне enum | да (→ `in-progress`) |
| 9 | Tags casing (lowercase аббревиатуры) | да |
| 10 | Tags inline-формат | да |
| 11 | Raw-refs в теле страницы | да |
| 12 | Пустые секции | нет (флаг) |
| 13 | Устаревшие записи в `index.md` (ведут на удалённые/переименованные страницы) | нет |

## Формат отчёта lint

`wiki/meta/lint-report-YYYY-MM-DD.md` (создаётся только если есть нерешённые проблемы):

```markdown
---
type: meta
title: "Lint Report YYYY-MM-DD"
updated: YYYY-MM-DD
---

# Lint Report: YYYY-MM-DD

## Сводка
- Просканировано страниц: N
- Найдено проблем: N
- Авто-исправлено: N
- Требует ревью: N

## Сироты
- [[Страница]]: нет входящих ссылок. Предложение: слинковать с [[X]] или удалить.

## Мёртвые ссылки
- [[Несуществующая]]: упомянута в [[Исходная]]. Предложение: создать заглушку или убрать ссылку.

## Пропущенные страницы
- "название": упомянуто в [[A]], [[B]], [[C]]. Предложение: создать idea-страницу.

## Устаревшие утверждения
- [[Страница]]: утверждение "X" может конфликтовать с [[Новый источник]].

## Пустые секции
- [[Страница]]: секция "## Применение" пуста.
```

---

## Конвенции для проверки

| Элемент | Конвенция | Пример |
|---|---|---|
| Имена файлов | Title Case с пробелами | `Reward Model.md` |
| Папки | lowercase | `wiki/ideas/`, `wiki/entities/` |
| Теги | аббревиатуры заглавными, обычные слова с заглавной первой буквы | `ML`, `RL`, `Alignment`, `Optimization` |
| Wikilinks | точно совпадают с именем файла | `[[Reward Model]]` |
| `[[raw/...]]` | без расширения `.md`, только в `sources` frontmatter | `[[raw/RLHF]]` |

Имена файлов уникальны по всему vault — wikilinks без путей работают только при уникальности.

---

## Стиль письма (флаг, не auto-fix)

Помечать страницы, нарушающие style guide:

- Не декларативное настоящее время ("X в основном делает Y" вместо "X делает Y")
- Отсутствуют ссылки на entity-страницы там, где сделаны утверждения о фактах из источников
- Неопределённость не помечена явно

---

## Перед автоисправлением

В режиме auto-fix (вызов из ingest) — правки применяются молча, в конце выводится сводка.

В режиме full audit (`/lint`) — сначала отчёт, потом вопрос: "Применить auto-fix сейчас?".

**Никогда не править автоматически:**
- удаление страниц-сирот (могут быть намеренно изолированы)
- разрешение противоречий (требует человеческого суждения)
- объединение дублирующих страниц
- содержание (только структура и frontmatter)
