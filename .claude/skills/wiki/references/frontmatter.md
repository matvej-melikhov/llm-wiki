# Схема frontmatter

Каждая wiki-страница начинается с YAML frontmatter. Только плоский YAML, без вложенности (этого требует Obsidian Properties UI).

Title не дублируется в frontmatter — он берётся из имени файла. H1 в теле страницы остаётся читаемым заголовком.

---

## Универсальные поля

Обязательны на каждой содержательной странице:

```yaml
---
type: <idea|entity|question|domain|meta>
created: 2026-04-29
updated: 2026-04-29
tags:
  - <Тематический-Тег>
status: <evaluation|in-progress|ready>
domain:
  - "[[Domain Page]]"
related:
  - "[[Другая страница]]"
sources:
  - "[[raw/articles/source-file.md]]"
---
```

### Значения `status`

Допустимые **только три**:

- `evaluation` — страница на оценке: создана, но содержание ещё проверяется или дорабатывается
- `in-progress` — активно развивается, есть реальный контент, но не завершена
- `ready` — готова, исчерпывающая, хорошо слинкована

Никаких других значений (`stable`, `draft`, `done` и т.д.) быть не должно. Если синтез не уверен — ставь `evaluation`.

### Поле `domain`

Список wikilinks на страницы из `wiki/domains/`. Создаёт явные рёбра в графе Obsidian — domain-страницы становятся узлами кластеризации. Может быть пустым списком: `domain: []`.

**Порядок: от частного к общему** (convention, не enforced). Если у страницы несколько доменов, перечисляй их **от самого специфичного к самому широкому**. Например, для страницы про PPO:

```yaml
domain:
  - "[[Reinforcement Learning]]"   # узкий, специфичный
  - "[[Machine Learning]]"          # широкий, зонтичный
```

Зачем: первый домен в списке считается primary classification страницы. Это используется в knowledge map для раскраски точек. Без соглашения две одинаковые страницы могут получить разные цвета только из-за порядка ввода.

Это редакторское решение автора — машина не может его автоматически вывести (например, через counts: новый широкий домен начнёт с 1 страницы и эвристика сочтёт его узким). Lint **не enforce'ит** этот порядок. Если видишь на карте знаний что страница раскрашена «не туда» — проверь порядок в её frontmatter.

Domain-страницы сами имеют себя в `domain:` (self-reference) — нужно для того, чтобы они раскрашивались на knowledge map в свой цвет.

### Поле `tags`

Тематические теги. Не дублируй тип страницы (тип уже хранится в `type`). Используется для фильтрации и для триггера предложений domain-страниц.

**Регистр тегов:**
- Аббревиатуры — **заглавными**: `ML`, `RL`, `NLP`, `RLHF`, `LLM`, `GAN`, `VAE`, `LoRA`.
- Обычные слова — **с заглавной первой буквой**: `Alignment`, `Reinforcement-Learning`, `Optimization`.
- Никаких lowercase-аббревиатур (`ml`, `rl`) — это шум и снижает читаемость в Properties UI.

## Тип-специфичные поля

### idea

Концепция, механизм, теория, паттерн.

```yaml
aliases:
  - "альтернативное название"
  - "аббревиатура"
```

### entity

Именованный объект реального мира.

```yaml
entity_type: person       # person | organization | product | repository | place | paper | model | dataset
role: ""
```

Поле `status` для entity **не используется** — entity это факт, у неё нет состояния готовности.

### question

Сохранённый ответ на конкретный вопрос.

```yaml
question: "Оригинальный запрос"
answer_quality: solid     # draft | solid | definitive
```

### domain

Навигационный хаб (Map of Content) для области знаний. Дополнительных полей не требуется — имя файла и `tags` определяют домен.

### meta

Служебная страница (index, log, cache, summary, dashboard, lint-отчёт). Минимальный frontmatter:

```yaml
---
type: meta
title: ""
updated: 2026-04-29
---
```

Meta — единственный тип, где допустимо поле `title` (для отображаемого названия служебной страницы вроде "Кэш контекста").

## Правила

1. Только плоский YAML. Никаких вложенных объектов.
2. Даты как строки `YYYY-MM-DD`, не ISO datetime.
3. Списки в формате `- элемент`, не inline `[a, b, c]`.
4. Wikilinks внутри YAML обязательно в кавычках: `"[[Страница]]"`.
5. Поля `related`, `sources`, `domain` содержат wikilinks, а не сырые URL.
6. Поле `updated` обновляется при каждом изменении контента страницы.
7. Поле `domain` ссылается только на страницы из `wiki/domains/`.
8. Поле `sources` ссылается только на файлы из `raw/`. Эти ссылки **не дублируются в теле страницы**.
